# ワイズマンシステムSP 技術仕様書

> 本ドキュメントは公式インストール手順書（製品250311-101）、実環境スクリーンショット、
> Web調査から判明した事実をまとめたもの。RPA開発の基礎資料として使用する。

## 1. システム構成

```
┌─────────────────────────────┐
│ ワイズマンASPサーバー         │  ← ワイズマン社がクラウドで運用
│ (*.wiseman.ne.jp)            │
└──────────▲──────────────────┘
           │ インターネット（HTTPS想定）
┌──────────┴──────────────────┐
│ クライアントPC (Windows)     │
│                              │
│  .NETネイティブアプリ         │  ← 各PCにインストール
│  (WinForms / MDI構成)        │
│                              │
│  USBキー or ライセンスID認証  │
└─────────────────────────────┘
```

**「ASP」の意味**: Application Service Provider = サーバーのホスティング形態。
クライアント側はブラウザではなく**.NETネイティブアプリ**をインストールして使用する。

## 3. UI構造（実環境スクリーンショットより）

### ウィンドウ階層（MDI）
```
親ウィンドウ: 「通所・訪問リハビリ管理システム SP(ケア記録) [施設名]」
  └─ 子ウィンドウ: 「ケア記録集計表」
       ├─ ボタン: [ヘルプ] [印 刷] [閉じる]
       ├─ コンボボックス: レイアウト選択 (01 [バイタル記録表])
       ├─ コンボボックス: 対象年月 (令和08年03月)
       ├─ ボタン: [<] [今月] [>]（月ナビゲーション）
       ├─ コンボボックス: 利用者条件 (サービス種類)
       ├─ チェックボックス群: サービス種類
       │    ☑通所リハ ☑予防通所リハ ☑訪問リハ ☑予防訪問リハ
       ├─ ボタン: [内訳]
       ├─ ラジオボタン: 自動集計項目表示位置 (左・上部 / 右・下部)
       ├─ ボタン: [全面表示]
       ├─ コンボボックス: 絞込み (全利用者)
       ├─ コンボボックス: 表示順 (フリガナ)
       ├─ ラジオボタン: [昇順] [降順]
       ├─ 表示: 絞込 215 人/総人数 215 人
       └─ データグリッド: ケア記録集計表
            ├─ ヘッダー行: 利用者名 | ケア記録項目 | 1〜31(日付) | 件数 | 平均 | 最大 | 最小
            ├─ データ行: 赤松 利枝 | 体温 (℃) | 36.1 | ... | 36.0
            │            赤松 利枝 | 脈拍 (回/分) | 87 | ... | 78.0
            │            赤松 利枝 | 血圧上 (mmHg) | 129 | ... | 129.0
            │            赤松 利枝 | 血圧下 (mmHg) | 80 | ... | 80.0
            └─ セル色: 赤ハイライト = 異常値

親ウィンドウ右側パネル:
  ├─ ボタン: [一覧表示] [表示項目]
  ├─ 日付ナビゲーション: 03月09日 [▼] [月]
  └─ 利用者リスト: 利用受付（始日/終了日）、介護保険証番号
```

## 5. データ抽出方式

### 方式A: CSV出力（印刷機能経由）
- [印 刷] ボタン → 出力形式選択 → ファイル保存ダイアログ
- 文字エンコーディング: Shift-JIS想定（要確認）

### 方式B: データグリッド直接読取（pywinauto）
- グリッドコントロールの行・列をプログラム的に読取
- 色情報（異常値ハイライト）も取得可能な場合がある

## 6. 認証・セキュリティ

| 項目 | 内容 |
|------|------|
| USBキー認証 | 物理USBドングルをPCに挿入。5台なら5本必要 |
| ライセンスID認証 | USBキー不要の代替認証方式（本クライアントは未使用） |
| アプリ内ログイン | **なし（確定 2026-04-08）**。USBドングル認証のみで、アプリ起動後は直接メインウィンドウが表示される。ユーザー名/パスワード入力画面は存在しない。詳細は ADR-007 参照。 |
| セッション | 一定時間操作なしでタイムアウトの可能性（要実機確認） |

## 7. 既知の制約・注意事項

| 項目 | 内容 | 出典 |
|------|------|------|
| USBキー紛失 | 一時的にシステム使用不可 | Web調査 |
| 操作の複雑さ | 「都度終了しないと次の画面に進めない」 | ユーザー口コミ |
| UI更新頻度 | 「10年以上前から中身の変化なし」 | ユーザー口コミ |
| ウイルスバスター | クラウド Ver 17.8で修正プログラム適用が必要な場合あり | インストール手順書 p.1 |
| Defender除外 | 初期設定ツールでDefender除外設定を自動実行 | インストール手順書 p.5 |
