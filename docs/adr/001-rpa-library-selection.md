# ADR-001: RPAライブラリの選定

## ステータス
Revised (2026-03-23) — 実環境スクリーンショットに基づく最終決定

> **経緯**:
> - 初版 (2026-03-22): pywinauto採用 → Web調査でASP型と誤認しPlaywrightに変更
> - 改訂 (2026-03-23): クライアント実環境のスクリーンショットにより**ネイティブWindowsデスクトップアプリ**と確定。pywinautoに回帰

## コンテキスト
ワイズマンシステムSPは**ASP型サービスだが、クライアントは.NETネイティブWindowsデスクトップアプリ**である。

### システム構成（公式インストール手順書より確定）
- **サーバー側**: ワイズマン社がクラウドでホスティング（ASP = Application Service Provider）
- **クライアント側**: 各PCに`.NETネイティブアプリ`をインストールして使用（ブラウザは使わない）
- **インストール先**: `C:\Users\{User}\AppData\Local\Programs\WISEMAN\WISEMANVSYSTEM\`
- **ランタイム**: .NET Framework 3.5（インストール時に自動有効化）
- **認証**: 「USBキー認証」または「ライセンスID認証」を選択
- **起動**: デスクトップショートカット「ワイズマンASPサービス起動」からexeを起動

### UI特性（実環境スクリーンショットより確認）
- **ウィンドウ形式**: MDI（Multiple Document Interface）— 親ウィンドウ内に子ウィンドウが開く
- **タイトルバー**: `通所・訪問リハビリ管理システム SP(ケア記録) [施設名]`
- **UIフレームワーク**: .NET WinForms（.NET Framework 3.5）
- **コントロール**: ボタン、コンボボックス、チェックボックス、ラジオボタン、データグリッド（色付きセル含む）

> **学んだこと**: 日本の業務ソフトで「ASP」は「サーバーのホスティング形態」を指し、
> クライアント側がブラウザベースとは限らない。ワイズマンはネイティブ.NETアプリを
> 各PCにインストールし、ワイズマン社のASPサーバーに接続するシンクライアント型構成。

## 検討した選択肢

### A. pywinauto（採用 — 主エンジン）
- Windows UI Automation / Win32 APIによるデスクトップアプリ自動化
- アクセシビリティベースのセレクタ（コントロール名、タイプ、AutomationID）
- MDIウィンドウの階層的アクセスに対応
- WinForms/.NET/VB6/Delphi等の標準コントロールに強い

### B. PyAutoGUI（採用 — フォールバック）
- スクリーン座標/画像マッチングベースの操作
- pywinautoで到達不可のカスタム描画コントロール用
- データグリッドのセル選択等でピクセル座標が必要な場合のバックアップ

### C. Playwright / Selenium
- Webブラウザ自動化ツール
- ネイティブデスクトップアプリには**非対応** → **不採用**

### D. UiPath / Power Automate
- 商用RPAツール。高機能だがライセンスコスト大 → **不採用**

## 決定
**pywinautoを主エンジン、PyAutoGUIをフォールバックとして採用する。**

## 理由

### 1. ワイズマンのUIに最適
実環境で確認されたUI要素（ボタン、コンボボックス、チェックボックス、ラジオボタン、グリッド）はすべてWindows標準コントロールであり、pywinautoのUI Automationバックエンドで高精度に操作可能。

### 2. MDI対応
```python
# ワイズマンのMDI構造に合わせた操作例
app = Application(backend="uia").connect(title_re=".*ワイズマン.*")
main_window = app.window(title_re=".*管理システム SP.*")
child = main_window.child_window(title="ケア記録集計表")
child.wait("visible", timeout=10)
```

### 3. データグリッドの読取
```python
# グリッドからデータ抽出
grid = child.child_window(control_type="DataGrid")
for row in grid.children(control_type="DataItem"):
    cells = row.children(control_type="Custom")
    # 利用者名, 項目名, 日付別データ...
```

### 4. 帳票出力（CSV/印刷）の操作
```python
# 印刷ボタン → CSV出力ダイアログ
child.child_window(title="印 刷").click()
save_dialog = app.window(title="名前を付けて保存")
save_dialog.Edit.set_text("C:\\export\\report.csv")
save_dialog.child_window(title="保存").click()
```

### 5. 補完戦略
pywinautoで操作困難な要素（カスタム描画グリッドセル等）には:
1. まずUI Automationバックエンドで試行
2. 失敗時はWin32バックエンドに切替
3. それでも不可能な場合はPyAutoGUIで画像マッチング/座標クリック

## ワイズマン固有の操作パターン（実画面より）

### 画面構成
```

│ 通所・訪問リハビリ管理システム SP(ケア記録) [施設名]   │ ← 親ウィンドウ
│ ┌─────────────────────────────────────────────────┐ │
│ │ ケア記録集計表                                    │ │ ← 子ウィンドウ(MDI)
│ │ [ヘルプ] [印 刷] [閉じる]                        │ │ ← 操作ボタン
│ │                                                  │ │
│ │ レイアウト: [01▼] [バイタル記録表]                 │ │ ← コンボボックス

```

### 操作シーケンス（PoC想定）
1. アプリ起動 → USBドングル認証 → ログイン画面
2. ユーザー名/パスワード入力 → メインメニュー
3. メニュー遷移: ケア記録 → 集計表
4. 条件設定: 対象年月、サービス種類
5. データグリッドからデータ読取 or CSV出力
6. ファイル保存ダイアログでCSV保存先指定

## 結果
- `src/wiseman_hub/rpa/` にRPAモジュールを配置
- `rpa/base.py`: 抽象インターフェース（Windows実装 + macOSモック）
- `rpa/login.py`: ログイン自動化
- `rpa/navigator.py`: MDIメニュー遷移
- `rpa/export_csv.py`: CSVエクスポート（印刷→保存ダイアログ操作）
- `rpa/grid_reader.py`: データグリッドの直接読取
- `rpa/window_utils.py`: ウィンドウ検索・待機ヘルパー
- セレクタ情報（コントロール名、AutomationID等）はTOML設定で外部化
- macOS開発環境ではモック実装でテスト
